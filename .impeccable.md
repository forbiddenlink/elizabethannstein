# Impeccable — Design Source of Truth

> Persistent design context for elizabethannstein portfolio. Every design/UI decision in this repo
> should trace back to this file. Generated via `teach-impeccable` 2026-06-29.

## Design Context

### Users

Recruiters and hiring engineers evaluating Liz as a candidate. Context: fast scanning — they're
deciding within seconds whether to dig deeper. Site doubles as the proof-of-craft: the visual
execution is itself a portfolio piece. Secondary audience: hiring managers at creative/product
companies who expect aesthetic sensibility alongside technical ability.

### Brand Personality

Distinctive, precise, cinematic. The site should communicate "senior engineer who also cares
about craft" — technically authoritative, not corporate-sterile. The "galaxy" metaphor is the
thread: each project category (Enterprise, AI, Full-Stack, DevTools, Design/Creative,
Experimental) is its own sector, with its own color. The site is an atlas of that work, not
a static resume.

Emotional target: **arresting + credible.** Not playful, not hype, not generic-SaaS.

### Aesthetic Direction

- Pure dark base (`#000000` / `#0a0a0a` elevated), white type, achromatic neutral accent.
  Color enters only through the galaxy palette — six category-specific hues used precisely.
- Glassmorphism throughout: `backdrop-filter: blur(20px) saturate(150-180%)` with near-black
  tinted backgrounds. Film grain overlay at `opacity: 0.042` for depth (`.grain-film`).
- Motion: GSAP for magnetic/tilt interactions, CSS keyframes for ambient (starfield drift,
  aurora, gradient flow). `prefers-reduced-motion` respected site-wide.
- Space-Grotesk (sans) + JetBrains Mono — deliberate departure from Inter/Syne defaults.

**Anti-references (must NOT look like):**

1. Generic AI SaaS template — gradient purple blob on white, Inter, identical stat grids.
2. Over-designed noise — the depth effects (grain, glass, starfield) must read as editorial,
   not gimmicky.
3. Resume-in-a-box — surfaces lived-in work (shipped URLs, outcomes), not a skills checklist.

### Design Principles

1. **Motion earns its place.** GSAP magnetic/tilt on `MagneticButton`, GalaxyCursor, WarpTransition
   page wipe, ScrollReveal reveals — all serve orientation or delight, not pure decoration.
   All respect `prefers-reduced-motion`.
2. **Galaxy palette = semantic law.** Each of the 6 galaxy colors maps to a project category.
   They are not interchangeable decorations. Use them to orient; never mix categories.
3. **Achromatic base, chromatic accent.** Core neutral system (surface-1 through surface-4,
   white text with opacity tiers) carries the UI. Color lives in the galaxy chips,
   progress bars, and project-specific `--case-accent` variable.
4. **Glass is the elevation model.** No traditional shadows for depth — use glassmorphism +
   border + slight background shift. `--color-surface` (#0a0a0a) → `--color-elevated` (#111111)
   is the only step-up.
5. **44px touch targets everywhere.** `.btn` min-height: 2.75rem, header nav links are
   `min-h-11`. Non-negotiable for recruiter-on-phone scenario.

### Accessibility

Bar: **best-effort, with real wins shipped.** `prefers-reduced-motion` implemented site-wide
(grain-film hidden, all keyframes collapsed, hover transforms stripped). Focus-visible: white
outline 2px + 4px glow for interactive elements. 44px touch targets on nav and buttons.
Color is never the sole carrier of meaning — galaxy labels accompany galaxy dots.

Color contrast is the main open gap: white/70% (#b3b3b3 equivalent) on pure black passes AA
for large text but is marginal for body. Investigate before any body-copy repalette.

## Current System (as-built reference, 2026-06-29)

- **Stack:** Next.js (App Router), React, TypeScript, Tailwind CSS v3 (config-based, not v4

## Shadows / Elevation

Shadow tokens are intentionally absent from `:root` — the comment in globals.css reads
"Shadows removed — elevation via color shift + border only."

Elevation is expressed through:
- Background shift: `--color-surface` (#0a0a0a) → `--color-elevated` (#111111)
- Glass: `backdrop-filter: blur(20px) saturate(150-180%)` + `border: 1px solid rgba(255,255,255,0.1)`
- Glass-panel full shadow: `0 8px 32px rgba(0,0,0,0.4), inset 0 1px 0 rgba(255,255,255,0.1)`
- Project-modal galaxy glows: `0 0 60px rgba(<galaxy-rgb>,0.14), 0 8px 32px rgba(0,0,0,0.5)`
- Card hover lift: `translateY(-2px)` + border brightening (no shadow added)

The two tailwind shadow tokens (`shadow-glow-purple`, `shadow-glow-blue`) are defined in
the config but their CSS var values are not defined in `:root`. TODO: resolve or remove.

## Component Conventions

### Buttons

- **Base class:** `.btn` — `min-height: 2.75rem (44px)`, `padding: 0 1.25rem`, `font-size: 0.875rem`, `font-weight: 500`, `border-radius: var(--radius-xl)`, `transition: all 0.2s var(--ease-out-expo)`
- **Primary (`.btn-primary`):** `bg: --color-accent (#f0f0f0)`, `color: #000000`
- **Secondary (`.btn-secondary`):** `bg: --color-surface`, `border: --color-border`, `color: --color-text-primary`
- **Ghost (`.btn-ghost`):** transparent, `white/70`, tighter padding `0 0.75rem`
- **Galaxy variants:** `.btn-galaxy-{enterprise|ai|fullstack|devtools|creative|experimental}` — solid galaxy color fill, hover at 85% opacity
- **MagneticButton component:** GSAP-powered, `strength: 0.4`, `tiltStrength: 15`, spring return (`elastic.out(1, 0.3)`), scale-up on hover (`1.05`, `back.out(1.7)`). `glowOnHover` adds `.magnetic-glow` (white border, `rgba(255,255,255,0.25)`). Reduced-motion: hover only, no position/tilt.
- **Ripple:** `.btn-ripple` class adds radial-gradient `::after` ripple on `:active`

### Cards / Panels

- **`.glass-card`:** `bg rgba(10,10,10,0.55)`, `blur(20px) saturate(150%)`, `border: 1px solid --color-border`. Shimmer sweep on hover via `::before`.
- **`.glass-panel`:** `bg rgba(0,0,0,0.6)`, `blur(20px) saturate(180%)`, `border rgba(255,255,255,0.1)`, `box-shadow: 0 8px 32px rgba(0,0,0,0.4) + inset top highlight`, `border-radius: --radius-xl`
- **`.glass-panel-subtle`:** `bg rgba(0,0,0,0.4)`, `blur(16px)`, `--radius-lg`
- **`.card-hover-lift`:** `translateY(-2px)` + background/border brightening on hover
- **`.surface-{1-4}`:** white-alpha surfaces at 0.02 / 0.05 / 0.08 / 0.12

### Navigation

- **SiteHeader:** fixed, `z-50`, `bg-[#0a0a0a]/95`, bottom border from active galaxy theme.
- Active nav link: galaxy-tinted `bg-{color}/20 + border + text-{color}-100`. Underline via
  absolute `h-0.5` bar at bottom of link.
- Inactive nav link: `text-white/60`, hover → `text-white` + galaxy bg tint.
- Logo: concentric circles (purple→indigo gradient layers). `scale-110` on group hover.
- "Open to work" pill: `bg-emerald-500/10 border-emerald-500/25 text-emerald-400`, animated ping dot.
- Galaxy-sector label badge: `hidden xl:inline-flex`, pill shape, themed to current route.

### Background / Ambient

- **Starfield:** `.starfield-bg` — fixed, CSS radial-gradient dots on `background-size: 350px`, two layers drifting at different speeds (60s / 90s).
- **Film grain:** `.grain-film` — SVG noise tile at `opacity: 0.042`, `mix-blend-mode: overlay`. Hidden under `prefers-reduced-motion`.
- **Aurora:** `.case-study-aurora` — dual elliptical radial gradients using `--case-accent` color, 42s drift, `mix-blend-mode: screen`.
- **Custom cursor:** `GalaxyCursor` component (canvas or pointer-follower). Standard cursor on `pointer: coarse`.

### Form Inputs

- **`.input-glass`:** `bg: --color-surface`, border: `--color-border`, `--radius-md`, `0.75rem 1rem` padding, placeholder: `--color-text-tertiary`. Focus: border white/40. Hover: border white/20.

### GSAP Usage (MagneticButton)

- Hover enter: `scale(1.05)`, `ease: 'back.out(1.7)'`, 0.4s
- Hover move: `x/y` offset × strength (0.4), `rotationX/Y` ± `tiltStrength` (15°), `ease: 'power2.out'`, 0.3s
- Hover leave: reset all, `ease: 'elastic.out(1, 0.3)'`, 0.6s
- Reduced motion: skip position/tilt entirely; hover state only
